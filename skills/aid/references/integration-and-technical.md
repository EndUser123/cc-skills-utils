## Constitutional Notes

When using aid for refactoring analysis:
1. **READ-BEFORE-WRITE**: Always explore existing code before recommending changes
2. **Truthfulness**: Report findings accurately, don't sugarcoat technical debt
3. **Value Maximization**: Capture all valuable insights from analysis
4. **Singular Dev Authority**: Recommend solo-dev appropriate patterns, not enterprise bloat

The refactoring prompts generated by aid include:
- Evidence-based findings (not speculation)
- Specific file locations and line numbers
- Concrete before/after examples
- Incremental improvement phases
- Rollback strategies
