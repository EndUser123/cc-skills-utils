# SAR Help

SAR is CWO-Compliant sub-agent coordination for complex tasks.

## Purpose

SAR Help documentation for CWO-compliant sub-agent coordination in complex task execution.

## Project Context

### Constitution/Constraints
- CWO (Concurrent Workflow Orchestrator) compliant
- Sub-agent coordination for complex tasks
- Evidence-based validation required

### Technical Context
- Workflow: Constitutional Pre-Check → Plan Enhancement → Advisory Analysis → Task Decomposition → Sub-Agent Integration → Evidence-Based Validation → Knowledge Integration

### Architecture Alignment
- Integrates with `/sar-inst` for SAR implementation
- Works alongside `/cwo` for workflow orchestration
- Suggests `/orchestrator` for multi-agent coordination

## Your Workflow

1. **Constitutional Pre-Check**: Validate against CSF NIP principles
2. **Plan Enhancement**: Refine approach with constitutional constraints
3. **Advisory Analysis**: Analyze requirements and constraints
4. **Task Decomposition**: Break into atomic subtasks
5. **Sub-Agent Integration**: Coordinate specialist sub-agents
6. **Evidence-Based Validation**: Verify all outputs with evidence
7. **Knowledge Integration**: Store learnings in CKS

## Validation Rules

### Prohibited Actions
- Do NOT skip constitutional pre-check
- Do NOT proceed without evidence-based validation
- Do NOT use sub-agents for trivial (reversibility <1.5) tasks

### Required Evidence
- All claims must cite source (file:line, output, etc.)
- Validation results must be shown
- Sub-agent outputs must be aggregated
