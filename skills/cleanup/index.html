Two-Phase Approach

1
Phase 1: Source Code Analysis ▼
Principle: Fix the source, not the symptom.
Run cleanup.py --dry-run to scan for violations
Search for code generating violations: coverage configs, test enforcers, data misplacement
Display source problems with file path, description, suggested fixes
Fix source code first — then re-run /cleanup
No creating code found? Violations are likely manual or dynamic. Proceed to Phase 2 but still fix the source if discovered during cleanup.
cleanup.py --dry-run → violations → source_trace → source_fix OR "manual, proceed"
2
Phase 2: Violation Cleanup ▼
Review displayed violations with reasons and suggested actions
Approve cleanup session when prompted
For each violation choose: [m]ove · [d]elete · [s]kip · [q]uit
View cleanup summary and final compliance verification
ActionWhat it does
[m]oveMove to suggested location
[d]eleteDelete the file or directory
[s]kipSkip this item
[q]uitExit cleanup
violations → interactive_review → [m|d|s|q] → summary → verify

Validation Rules

●
Prohibited Actions ▼
Do NOT cleanup without reviewing violations first
Do NOT fix violations without addressing source code that created them
Do NOT delete directories without checking for active references
●
Safety Features ▼
Reference checking before deletion — prevents deleting actively-used directories
Per-file confirmation required in interactive mode
Summary + verification after cleanup
Dry-run mode for preview without changes
