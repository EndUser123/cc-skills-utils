Operation Logging Format

## Instance Tracking

When operating with 5+ concurrent instances, include in logs:

- Instance identifier (session ID or UUID)
- Timestamp of operation
- File(s) being modified
- Operation type (read, write, delete, etc.)
- Git status before operation

## Format Example

[INSTANCE-abc123] 2025-01-07T12:00:00 MODIFY file.py - Status: clean
